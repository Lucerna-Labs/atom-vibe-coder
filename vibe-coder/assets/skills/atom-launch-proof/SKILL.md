# Atom Launch Proof

Launch the product through the same path an operator uses. Capture startup
output, confirm the process remains alive without panic, wait for the first
usable screen, and capture a screenshot.

Then drive a real click or typed input. Capture the input event, its complete
Spiderweb L0-L3 route, the runtime result, and a changed rendered-state hash.
A screenshot by itself proves only that one frame drew and cannot pass.

Reconfirm the definition of done, zero deferred debt, zero COUPLE markers, and
a clean completion enforcer. Only this deterministic pass permits the planner
to mark the build complete.
