# Atom Build Intake

Pin down purpose, observable user behaviors, UI choice, persistence choice,
external boundaries, local/cloud siting, explicit exclusions, and an observable
definition of done. Use the operator request, Wiki Graph evidence, and
scratchpad only as data. Ask a question only when a missing answer cannot be
derived safely from the request or existing project contract.

Do not design crates, freeze message types, write implementation, or pick a
provider in this step.

The step passes only when every requirements field is explicit, the record is
written as a hashed artifact, and the deterministic intake gate recomputes it.
